# لوحة تحكم RDAPify للمسؤولين

**الرابط:** https://rdapify.com/adify

لوحة تحكم RDAPify للمسؤولين هي واجهة إدارة قوية لإدارة التراخيص وإدارة المستخدمين ومراقبة المنصة.

## الوصول

### المصادقة مطلوبة

- **رمز المسؤول** — مطلوب لجميع عمليات الإدارة
- **المصادقة الثنائية** (2FA) — موصى به لأسباب أمنية
- **قائمة عناوين IP البيضاء** — طبقة أمان إضافية اختيارية

### تسجيل الدخول

```
POST https://api.rdapify.com/admin/login
{
  "email": "admin@rdapify.com",
  "password": "your-password",
  "2fa_code": "123456" // إذا تم تفعيل 2FA
}
```

الاستجابة:
```json
{
  "session_token": "sess_xxxxx",
  "expires_at": 1710895999999
}
```

## الميزات

### 1. إدارة التراخيص

#### عرض التراخيص
- تصفية حسب الخطة (Pro و Team و Enterprise و Trial)
- البحث باسم المنظمة
- عرض مفتاح الترخيص وانتهاء الصلاحية
- مراقبة استخدام المقاعد
- التحقق من حالة الإلغاء

#### إصدار تراخيص جديدة
- إنشاء التراخيص برمجياً
- تعيين تواريخ انتهاء صلاحية مخصصة
- تكوين حدود المقاعد
- إنشاء مفاتيح تجريبية (30 يوماً)

#### إدارة التراخيص
- إلغاء التراخيص النشطة
- تمديد تواريخ انتهاء الصلاحية
- التحويل بين المنظمات
- عرض مسار التدقيق

### 2. إدارة المستخدمين

#### عرض المستخدمين
- سرد جميع المستخدمين المسجلين
- تصفية حسب الخطة أو الحالة
- البحث بالبريد الإلكتروني/المنظمة
- مراقبة تاريخ آخر تسجيل دخول

#### إدارة الحسابات
- إلغاء تفعيل/إعادة تفعيل المستخدمين
- إعادة تعيين كلمات المرور
- تحديث معلومات الفواتير
- إدارة إعدادات 2FA

### 3. التحليلات والمراقبة

#### المقاييس في الوقت الفعلي
- عدد التراخيص النشطة
- الإيراد الشهري (MRR)
- معدل الخروج
- اتجاهات النمو

#### تحليلات الاستعلام
- الطلبات يومياً
- متوسط وقت الاستجابة
- معدلات الخطأ حسب المنطقة
- أكثر النطاقات المستعلم عنها

#### نشاط المستخدم
- نشاط تسجيل الدخول
- أحداث إنشاء الترخيص
- سجلات استخدام API
- الأحداث الأمنية

### 4. Webhooks والتكاملات

#### إدارة Webhooks
- تسجيل نقاط نهاية webhook
- اختبار تسليم webhook
- عرض سجل التسليم
- إعادة محاولة webhooks الفاشلة

#### الأحداث المدعومة
- `license.created` — ترخيص جديد صادر
- `license.revoked` — تم إلغاء الترخيص
- `license.expired` — انتهت صلاحية الترخيص
- `subscription.updated` — تم تحديث الاشتراك
- `user.registered` — تسجيل مستخدم جديد

### 5. الفواتير والمدفوعات

#### لوحة معلومات الإيرادات
- الإيراد الشهري المتكرر (MRR)
- إجمالي العملاء
- تحليل الخروج
- اتجاهات الإيرادات

#### إدارة المدفوعات
- عرض معاملات Paddle
- تحميل الفواتير
- إدارة استرجاعات المبالغ
- عرض فشل المدفوعات

### 6. الأمان والامتثال

#### سجل تدقيق الأمان
- عرض جميع إجراءات الإدارة
- تتبع عنوان IP
- تسجيل الطابع الزمني
- سجل التغييرات

#### تقارير الامتثال
- طلبات تصدير بيانات GDPR
- سجلات حذف الحسابات
- سياسات الاحتفاظ بالبيانات
- حالة التشفير

### 7. العمليات الجماعية

#### إدارة جماعية
- اختيار تراخيص متعددة مرة واحدة
- تنفيذ إجراءات جماعية (إلغاء، تمديد)
- نوافذ تأكيد للأمان
- تتبع التقدم للدفعات الكبيرة

### 8. التصفية المتقدمة والتصدير

#### تصدير البيانات
- تصدير التراخيص والإيرادات والأمان
- صيغ متعددة: JSON، CSV
- تصفية بنطاق تاريخ مخصص
- رسائل بريد تقارير مجدولة

### 9. إدارة الفريق

#### فريق الإدارة
- دعوة أعضاء الفريق مع التحكم في الأدوار
- أنواع الأدوار: مسؤول عام، عارض
- تتبع الدعوات وآخر تسجيل دخول
- إزالة الوصول فوراً

### 10. إدارة مفاتيح API

#### الوصول البرمجي
- إنشاء مفاتيح API مسماة للأتمتة
- تعيين أذونات دقيقة (قراءة فقط، قراءة وكتابة)
- تكوين انتهاء الصلاحية (30 يوم، 90 يوم، سنة، لا ينتهي)
- إلغاء المفاتيح فوراً وتتبع الاستخدام

### 11. القواعل التلقائية والسير العمل

#### محرك الأتمتة
- نظام قواعل الحافز والإجراء
- المحفزات: انتهاء التراخيص، منتهية، شذوذ، غير نشطة
- الإجراءات: إرسال بريد، إلغاء، تمديد
- تفعيل/تعطيل القواعل فوراً

### 12. المراقبة المتقدمة

#### صحة النظام
- إحصائيات قاعدة البيانات (عدد الصفوف لكل جدول)
- زمن الاستجابة الخاص بـ API وحجم الطلبات
- تتبع حدود المعدل
- مراقبة الجلسات النشطة
- إحصائيات الـ cache

### 13. مركز الأمان

#### الكشف المتقدم عن التهديدات
- سجلات محاولات OTP الفاشلة
- النشاط المريب القائم على IP
- إدارة جلسات الإدارة النشطة
- مستويات شدة الشذوذ والتصفية
- وظيفة حجب IP

### 14. نظام الإخطارات

#### التنبيهات متعددة القنوات
- مفاتيح إخطارات البريد الإلكتروني (تم الإنشاء، منتهي الصلاحية، ملغى، شذوذ)
- تكامل خطاف Slack
- اختبار تسليم الإخطار
- تفضيلات التنبيه القابلة للتخصيص

## مرجع API

### المصادقة

جميع طلبات API تتطلب رمز جلسة صحيح:

```bash
curl -H "Authorization: Bearer {session_token}" \
  https://api.rdapify.com/admin/licenses
```

### نقاط النهاية

#### GET /admin/licenses

سرد جميع التراخيص مع المرشحات.

```bash
GET /admin/licenses?plan=pro&status=active&limit=50&offset=0
```

المعاملات:
- `plan` — تصفية حسب الخطة (pro و team و enterprise و trial)
- `status` — تصفية حسب الحالة (نشط و منتهي الصلاحية و ملغي)
- `organization` — البحث باسم المنظمة
- `limit` — النتائج لكل صفحة (افتراضي: 50)
- `offset` — إزاحة الصفحات

#### POST /admin/licenses

إنشاء ترخيص جديد.

```bash
POST /admin/licenses
{
  "plan": "pro",
  "organization": "My Company",
  "expiry_ms": 1710895999999,
  "max_developers": 5
}
```

#### DELETE /admin/licenses/{key}

إلغاء ترخيص.

```bash
DELETE /admin/licenses/RDAP-PRO-xxxxx
```

## استكشاف الأخطاء

### لا يمكن الوصول إلى لوحة المعلومات

1. تحقق من تسجيل بريدك الإلكتروني للإدارة
2. تحقق من رمز 2FA (إذا تم تفعيله)
3. امسح ذاكرة التخزين المؤقت وملفات تعريف الارتباط للمتصفح
4. جرب الوضع الخاص/الحاضنة
5. تواصل مع support@rdapify.com

### عدم ظهور الترخيص

1. تحقق من صيغة مفتاح الترخيص: `RDAP-{PLAN}-{payload}-{signature}`
2. تحقق من تطابق اسم المنظمة
3. تأكد من عدم انتهاء صلاحية الترخيص
4. حاول تحديث الصفحة

### عدم تسليم Webhook

1. تحقق من إمكانية الوصول إلى عنوان URL للنقطة النهائية علناً
2. تحقق من توقيع webhook في السجلات
3. اختبر باستخدام زر "إرسال حدث اختبار"
4. راجع سجل التسليم للحصول على رسائل الخطأ
