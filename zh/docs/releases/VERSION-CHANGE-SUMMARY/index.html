✅ تغيير رقم الإصدار من 0.1.4 إلى 0.1.2

التاريخ: 27 يناير 2026
الإصدار الجديد: 0.1.2
الإصدار السابق: 0.1.4 (لم يُنشر)
الحالة: ✅ مكتمل

🎯 السبب​

المشكلة​

  • آخر إصدار منشور على npm: 0.1.1 (25 يناير 2026)
  • الإصدارات 0.1.2 و 0.1.3 و 0.1.4 لم تُنشر أبدًا على npm
  • القفز من 0.1.1 إلى 0.1.4 يخلق فجوة في سجل الإصدارات

الحل​

  • تغيير الإصدار الحالي من 0.1.4 إلى 0.1.2
  • التسلسل الطبيعي: 0.1.1 → 0.1.2 ✅
  • الحفاظ على semantic versioning

- "version": "0.1.4",
+ "version": "0.1.2",

  • تحديث شعار الإصدار: v0.1.4 → v0.1.2

  • تحديث جدول الإحصائيات

🚀 الخطوات التالية​

1. دمج الفرع​

git checkout main
git merge fix/docs-build-issues
git push origin main

2. إنشاء Tag​

git tag -a v0.1.2 -m "Release v0.1.2: Interactive Playground"
git push origin v0.1.2

3. النشر على npm​

npm publish

أو انتظر GitHub Actions للنشر التلقائي عند push الـ tag.

4. التحقق من النشر​

npm view rdapify version
# يجب أن يظهر: 0.1.2

npm view rdapify versions
# يجب أن يظهر: [..., "0.1.1", "0.1.2"]

✅ الفوائد​

1. التسلسل الصحيح​

  • ✅ 0.1.0 → 0.1.1 → 0.1.2
  • ❌ 0.1.0 → 0.1.1 → 0.1.4 (فجوة)

2. Semantic Versioning​

  • الإصدار 0.1.2 يتبع المعيار
  • لا توجد فجوات في الأرقام
  • سهولة التتبع

3. npm Registry​

  • سجل نظيف بدون فجوات
  • المستخدمون يمكنهم رؤية التطور الطبيعي
  • لا ارتباك حول الإصدارات المفقودة

4. الوثائق​

  • جميع المراجع متسقة
  • لا تناقضات بين الكود والوثائق
  • سهولة الصيانة
